使用

以下是Anaconda常用命令的详解表格（覆盖初始化、环境管理、默认配置）：

命令 作用 示例
conda init <shell> 初始化conda环境（将conda命令添加到Shell配置） conda init bash（针对bash终端）
conda activate <env_name> 激活指定虚拟环境（默认环境为base） conda activate base（激活base环境）
conda config --set auto_activate_base <bool> 设置是否自动激活base环境 conda config --set auto_activate_base False（关闭自动激活）
conda create -n <env_name> python=<version> 创建新的隔离环境 conda create -n ml_env python=3.10（创建名为ml_env的Python3.10环境）
conda env list 列出所有已创建的环境（显示路径和名称） conda env list
conda install <package> 在当前环境安装指定包（支持版本指定） conda install numpy=1.26（安装numpy 1.26版本）
conda remove -n <env_name> --all 删除指定虚拟环境 conda remove -n ml_env --all（删除ml_env环境）

常见坑

  1. 安装后conda命令不可用：
    原因是安装时未选择“yes”初始化conda。解决方法：手动运行conda init，再执行source ~/.bashrc重启环境。

  2. 不想自动激活base环境：
    默认安装后会自动激活base环境，若需关闭，执行conda config --set auto_activate_base False，重新打开终端生效。

  3. 卸载不彻底导致残留：
    仅删除安装目录不够，需清理~/.bashrc或~/.zshrc中的# >>> conda initialize >>>代码块，否则重新打开终端仍会有conda提示。

  4. 多用户共享Anaconda：
    默认安装在用户目录（~/anaconda3），若要多用户共享，需安装到公共路径（如/opt/anaconda3），并修改权限为chmod -R 755 /opt/anaconda3，其他用户需运行conda init初始化自己的环境。
